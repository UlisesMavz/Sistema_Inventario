Visión General

Este documento describe la arquitectura técnica del Sistema de Gestión de Inventario. El sistema está diseñado para funcionar en un entorno local utilizando el paquete XAMPP en Windows, implementando algoritmos de ordenamiento y búsqueda desde cero para demostrar conocimientos de análisis de algoritmos.

Objetivo Principal: Desarrollar un sistema completo que implemente manualmente algoritmos de ordenamiento (Bubble Sort, Quick Sort) y búsqueda (Lineal, Binaria) sin usar funciones nativas, aplicando arquitectura MVC y API REST.

Patrones de Diseño

1. MVC (Model-View-Controller)

Propósito: Separar la lógica de negocio, la presentación y los datos.

  • Model: Producto, Usuario — Representan entidades del dominio
  • View: HTML/CSS/JS — Interfaz de usuario
  • Controller: ProductoController, AuthController — Coordinan operaciones

2. Singleton (Database)

Garantiza una única instancia de conexión a la base de datos, evitando múltiples conexiones innecesarias y mejorando el rendimiento.

3. REST API

Arquitectura de servicios web que usa HTTP para comunicación:

GET /api/productos.php → Obtener productos POST /api/productos.php → Insertar producto DELETE /api/productos.php → Eliminar producto GET /api/buscar.php → Buscar producto POST /api/ordenar.php → Ordenar productos

Decisiones de Diseño

¿Por qué MVC?

  • Separación de responsabilidades: Cada componente tiene un propósito claro
  • Mantenibilidad: Cambios en una capa no afectan a las demás
  • Testabilidad: Cada capa puede probarse independientemente
  • Escalabilidad: Fácil agregar nuevas funcionalidades
  • Trabajo en equipo: Diferentes desarrolladores en diferentes capas

¿Por qué API REST?

  • Desacoplamiento: Frontend y backend independientes
  • Reutilización: Misma API para web, móvil, desktop
  • Estándar: Protocolo HTTP ampliamente conocido
  • Escalabilidad: Fácil distribuir en múltiples servidores
  • Mejor UX: Actualizaciones sin recargar página completa

¿Por qué Implementación Manual de Algoritmos?

Objetivo Académico: Demostrar comprensión profunda de análisis de algoritmos, no solo usar funciones nativas de PHP como sort() o array_search().

Esto permite:

  • Contar operaciones reales y validar complejidades teóricas
  • Comprender el funcionamiento interno de los algoritmos
  • Comparar empíricamente diferentes enfoques (O(n²) vs O(n log n))
  • Aplicar conocimientos de la materia de Análisis de Algoritmos

Campo posicion en Base de Datos

Problema: SQL INSERT no garantiza orden específico de inserción.

Solución: Campo posicion que simula lista enlazada en BD relacional.

  • Permite inserción al inicio/final independiente del ID auto-incremental
  • Mantiene orden deseado por el usuario
  • Facilita operaciones de lista enlazada en contexto relacional
  • Índice en posicion para consultas eficientes

Algoritmos Implementados

Ordenamiento

Bubble Sort

Complejidad: O(n²)

  • Algoritmo simple e intuitivo
  • Útil para conjuntos pequeños
  • Implementado por precio y nombre
  • Optimización con bandera
Quick Sort

Complejidad: O(n log n)

  • Divide y Conquista
  • Muy eficiente en promedio
  • Implementado por precio y nombre
  • Recursivo con partición

Búsqueda

Búsqueda Lineal

Complejidad: O(n)

  • Recorrido secuencial
  • Funciona con datos desordenados
  • Implementado por código y nombre
  • Simple y universal
Búsqueda Binaria

Complejidad: O(log n)

  • Divide espacio a la mitad
  • Requiere datos ordenados
  • Implementado por código y nombre
  • Extremadamente eficiente
