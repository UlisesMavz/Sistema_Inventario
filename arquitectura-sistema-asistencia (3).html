Arquitectura del Sistema
de Asistencia QR

Documentación técnica completa del sistema académico con códigos QR — Un enfoque elegante para la gestión moderna de asistencia

Visión General

Este documento describe la arquitectura técnica del Sistema de Asistencia Académica mediante Códigos QR. El sistema está diseñado para funcionar en un entorno local utilizando el paquete XAMPP en Windows, gestionando asistencia, usuarios y reportes académicos.

Stack Tecnológico

Capa Tecnología Descripción
Servidor Web Apache Procesa las peticiones HTTP/HTTPS (vía XAMPP)
Lenguaje Backend PHP 7.4 / 8.x Ejecuta la lógica no autenticada y de negocio
Base de Datos MySQL / MariaDB Almacenamiento relacional de datos (Usuarios, Materias, Asistencias)
Frontend HTML5, CSS3, JS Interfaz nativa, diseños responsivos (Vanilla CSS/JS)
Librerías Externas FPDF Generación de reportes en PDF
Librerías Externas PHPQRCode Generación dinámica de códigos QR para asistencia

Arquitectura de Software

El sistema sigue un patrón modular basado en Page-Controller (Controlador por Página) con separación de lógica de datos en modelos (libs/).

Estructura de Capas

1
Capa de Presentación
  • Archivos .php en directorios como admin/, profesor/, alumno/
  • Manejan la lógica de recepción de peticiones (POST/GET)
  • Validación de sesión y renderizado de HTML
2
Capa de Lógica de Negocio
  • Ubicada en libs/
  • Clases como User.php, Materia.php, Asistencia.php
  • Encapsulan las consultas SQL y la lógica CRUD
3
Capa de Configuración
  • config/db.php: Singleton de conexión a BD usando PDO
  • Seguridad y portabilidad
  • assets/: Recursos estáticos (CSS, JS, Imágenes)

Modelo de Seguridad

Componentes de Seguridad

Autenticación por Sesión RBAC (Control de Acceso) Hash Bcrypt AES-256-CBC Sentencias Preparadas
  • Autenticación: Basada en $_SESSION de PHP
  • Autorización (RBAC): Validación de id_rol al inicio de cada script protegido
    • 1 = SuperAdmin
    • 2 = Admin
    • 3 = Profesor
    • 4 = Alumno
  • Protección de Datos:
    • Contraseñas: Hash password_hash() (Bcrypt)
    • Contraseñas Descifrables (SuperAdmin): Cifrado AES-256-CBC (openssl_encrypt) para recuperación
    • SQL Injection: Uso estricto de Sentencias Preparadas (PDO)

Especificaciones del Servidor de Producción

Sistema Operativo
  • Debian 13 (Trixie)
  • Kernel Linux 6.x
  • Arquitectura x86_64
Servidor Web
  • Apache2 (última versión estable)
  • Módulos: mod_rewrite, mod_ssl
  • Certificado SSL/TLS configurado
Base de Datos
  • MariaDB 10.11+
  • Compatible con MySQL
  • Optimizada para producción
Runtime
  • PHP 8.3 (FPM)
  • Extensiones: PDO, MySQLi, GD
  • Composer para dependencias
Seguridad
  • Firewall UFW configurado
  • Fail2Ban para protección
  • Actualizaciones automáticas
Respaldos
  • Cron jobs programados
  • Backup diario de BD
  • Snapshot de archivos

Requisitos de Ejecución

  1. XAMPP Instalado: Servicios Apache y MySQL iniciados
  2. Configuración de Base de Datos:
    Host: localhost User: root Pass: (Vacío por defecto en XAMPP) DB Name: sist_asistencia
  3. Permisos: Acceso de escritura en carpetas temporales para generación de QR/PDF
